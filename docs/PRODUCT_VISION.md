# PRODUCT_VISION.md

> This document defines the vision, goals, and long-term direction of the Lunar Core project.

---

## Purpose

Lunar Core is developed to provide a stable foundation for WordPress-based documentation websites.

The plugin focuses on data management, content structure, and Gutenberg Blocks to make creating and maintaining documentation easier, more consistent, and scalable over the long term.

---

## Vision

To become a WordPress plugin that provides a modern foundation for building fast, lightweight, and maintainable documentation websites.

The success of the project is measured not by the number of features it offers, but by how effectively it helps users manage content while remaining relevant as their websites grow.

---

## Core Values

The development of Lunar Core is guided by the following values:

- Content First
- Foundation First
- Quality Over Quantity
- WordPress Native
- Performance by Default
- Simplicity
- Maintainability
- Consistency
- Long-Term Thinking

---

## Target Users

Lunar Core is primarily designed for:

- Game documentation websites
- Wiki websites
- Knowledge bases
- Websites with complex content structures

However, the plugin can also be used for many other types of documentation websites.

---

## Long-Term Direction

The long-term development of Lunar Core will continue to focus on strengthening its foundation, including:

- Gutenberg Blocks
- Content structure
- Performance
- Accessibility
- Developer Experience
- Project documentation
- Long-term compatibility

New features will be considered only when they provide meaningful value to users and remain aligned with the project's primary goals.

---

## Definition of Success

Lunar Core is considered successful when it:

- remains lightweight and fast;
- is easy to maintain and extend;
- helps manage documentation at scale;
- follows WordPress standards;
- provides a stable foundation for long-term development.

---

## Non-Goals

Lunar Core is not intended to:

- become the plugin with the most features;
- add features simply to follow trends;
- replace WordPress core functionality;
- sacrifice performance for features that provide little real value;
- introduce unnecessary complexity.
